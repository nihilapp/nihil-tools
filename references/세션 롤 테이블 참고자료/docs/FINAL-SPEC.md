# TRPG 세션 정보 생성기 최종 설계

## 1. 앱 목적

이 앱은 완성된 세션 설계서를 만드는 도구가 아닙니다.  
GM이 세션 아이디어를 빠르게 떠올리도록 `의뢰 중심의 조합 카드`를 생성하는 도구입니다.

## 2. 화면에 표시할 정보

결과 화면에는 아래 정보만 표시합니다.

```txt
의뢰 대분류: 연맹 의뢰 / 민간 의뢰
의뢰 종류: 개척자 / 해결사 / 사냥꾼 / 민간 생활형 분류
의뢰인: 토큰 또는 역할명
의뢰 사유: 단어 토큰 배열
테마: 테마명
등장 예시 몬스터: <D&D 14 유형> 몬스터
```

다음 정보는 기본 출력에서 제외합니다.

```txt
후킹 포인트
진행 이유
비밀
장면 설계
문장형 의뢰 설명
상세 서사
```

## 3. 의뢰 대분류

### 연맹 의뢰

연맹 의뢰는 공식 공고입니다.  
의뢰인은 민간인이 아니라 세계의 미지를 추구하거나 탐사 체계에 연계된 존재입니다.

대표 의뢰인 계열은 다음과 같습니다.

```txt
연맹 행정·공고
탐사·기록
마법·마도 연구
드워프 기술·발굴
엘프 마나·정령 관측
던전·봉인·위험물 관리
```

연맹 의뢰 종류는 다음 3개입니다.

```txt
개척자
해결사
사냥꾼
```

`기밀`은 의뢰 종류가 아닙니다.  
`기밀`은 보안 등급 또는 특수 공고 플래그입니다.

### 민간 의뢰

민간 의뢰는 공식 연맹 공고 유형을 따르지 않습니다.  
민간인이 생계, 가족, 물건, 생활 피해, 작은 괴담처럼 자신들이 해결할 수 없는 일을 대신 부탁하는 구조입니다.

민간 의뢰 종류는 다음처럼 생활형 분류를 사용합니다.

```txt
생활 피해
생계 위협
실종·수색
물건 회수
짐승·마수 피해
길 안내·호위
농지·가축 문제
가게·상단 문제
작은 괴담
병자·약재 문제
이웃 갈등
비밀 부탁
```

## 4. 개척자 의뢰 별도 묶음

개척자 의뢰는 `앙그라의 안개권 공식 탐사 공고`로 별도 묶습니다.

개척자 의뢰 유형은 다음 5개입니다.

```txt
앙그라 경계 관측
앙그라의 못 탐색
토템 가디언 제압
안개권 지도 작성
안개권 생태·변이 조사
```

이 묶음은 일반 생활권 의뢰가 아니라 앙그라의 안개, 경계선, 못, 토템 가디언, 안개권 생태와 연결됩니다.

## 5. 사냥꾼 의뢰 재정의

사냥꾼은 문명권 내에서 몬스터 사냥과 위험 개체 대응을 맡는 계열입니다.  
던전 내부에서도 활동하고, 일반 필드에서도 활동합니다.

사냥꾼 의뢰의 대응 세트는 다음처럼 구분합니다.

```txt
던전 내부 세트
- 던전 일반 개체
- 던전 강화 개체
- 던전 우두머리 개체

일반 필드 잔향 생물 세트
- 잔향 개체
- 잔향 강화 개체
- 잔향 우두머리 개체

별도 세트
- 일반 짐승
- 일반 필드 몬스터
- 앙그라 안개권 위협 개체
```

전자의 3세트는 던전 내부의 일입니다.  
후자의 3세트는 일반 필드의 생물이 잔향에 오염된 경우입니다.

## 6. 의뢰 사유 규칙

의뢰 사유는 문장으로 작성하지 않습니다.  
반드시 단어 토큰 배열로 관리합니다.

좋은 예시는 다음과 같습니다.

```txt
[관측탑, 신호, 두절]
[가축, 야간, 실종]
[던전, 가디언, 광폭화]
[잔향, 생물, 우두머리]
```

나쁜 예시는 다음과 같습니다.

```txt
관측탑의 신호가 끊겨서 의뢰했다.
가축이 밤마다 사라져 생계가 위협받고 있다.
```

문장이 필요하다면 데이터가 아니라 UI 렌더링 단계에서만 조립합니다.

## 7. 등장 예시 몬스터 규칙

세션 생성 결과에서는 몬스터의 상세 이름을 만들지 않습니다.  
아래 D&D 14 유형 중 하나 이상을 골라 `<유형> 몬스터`로 표시합니다.

```txt
변이체
야수
천상체
구조체
드래곤
원소체
요정
악마체
거인
인간형
괴수
점액체
식물체
언데드
```

예시는 다음과 같습니다.

```txt
등장 예시 몬스터: 변이체 몬스터
등장 예시 몬스터: 야수 몬스터
등장 예시 몬스터: 구조체 몬스터
```
