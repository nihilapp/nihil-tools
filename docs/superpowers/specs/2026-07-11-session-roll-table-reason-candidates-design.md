# 세션 롤 테이블 사유 후보 재설계

## 목표

세션 롤 테이블의 공고 사유를 문장 조각이 아닌, 공고 맥락을 요약하는 짧은 단어 또는 복합어 후보로 구성한다. 생성 결과는 후보 중 서로 다른 1개 또는 2개를 표시한다.

## 범위

- 연맹 공고 85개와 민간 의뢰 12개, 총 97개를 대상으로 한다.
- 연맹 공고의 세계관·분류 근거는 `references/세션 롤 테이블 참고자료/자유 탐사 연맹 공고 유형.md`를 따른다.
- `references/세션 롤 테이블 참고자료/tables/league-notice-token-mapping.tsv`는 최신 85개 공고의 그룹·유형·역할·보안 정합성 검증에만 사용한다. 이 파일의 기존 토큰 세트는 후보 데이터로 재사용하지 않는다.
- 기존의 평탄한 `reasonTokenSets` 데이터와 이를 전제로 한 목록 문서는 제거한다.

## 데이터 모델

### 연맹 공고

- 17개 `noticeGroup`마다 공통 사유 후보 배열을 둔다. 각 배열은 해당 그룹의 현상, 위험, 수행 목적을 나타내는 후보를 20개 이상 가진다.
- 85개 `noticeType`마다 세부 사유 후보 배열을 둔다. 각 배열은 해당 유형만의 목표·대상·상황을 나타내는 후보를 12개 이상 가진다.
- 생성 후보는 `noticeGroup` 공통 배열과 `noticeType` 세부 배열을 합친 뒤, 동일 문자열을 제거한 배열이다. 따라서 각 공고가 사용할 수 있는 후보는 최소 30개다.
- 공통·세부 후보 간의 의미상 중복은 허용하되, 같은 문자열은 결과 후보 배열에 한 번만 남긴다.

### 민간 의뢰

- 민간 의뢰는 연맹의 공고 그룹 체계를 공유하지 않는다.
- 12개 `requestType`마다 독립 후보 배열을 30개 이상 둔다.
- 후보는 생활 피해, 실종, 회수, 호위, 괴담, 약재, 갈등 등 해당 의뢰의 맥락을 짧게 표현한다.

### 표현 기준

- 후보는 한 단어 또는 짧은 두세 어절 표현을 허용한다.
- 후보는 원문 문장을 기계적으로 쪼개지 않는다.
- TSV에 있는 기존 토큰 세트도 새 후보의 원본으로 사용하지 않는다.
- 후보는 공고에서 해결하거나 알아내야 할 대상, 현상, 위험, 목적을 표현한다. 예: `생활권 위협`, `우두머리 흔적`, `핵 회수`, `피난로 확보`, `룬 오염`.

## 생성 규칙

- 합쳐진 후보 배열에서 무작위로 1개 또는 2개를 선택한다.
- 1개와 2개는 같은 확률로 선택한다.
- 2개를 선택할 때 동일 후보는 표시하지 않는다.
- 표시 순서는 선택된 순서를 유지한다.

## 구현 경계

- 타입 이름은 후보의 실제 구조와 맞게 `reasonTokens` 또는 `reasonCandidates`처럼 단일 배열임을 드러낸다. `reasonTokenSets`는 사용하지 않는다.
- 공통 후보 병합과 중복 제거는 데이터 접근 함수에서 수행하고, Vue 컴포넌트는 완성된 후보 배열만 받아 선택한다.
- 공고 데이터와 민간 의뢰 데이터는 각각 현재의 데이터 파일 분리 구조를 유지한다.
- 기존의 원시 사유 단어 전수 목록 `docs/session-roll-table-reasons.md`는 새 후보 모델을 오해하게 하므로 삭제한다.

## 검증

- 모든 연맹 공고는 병합 후 후보 수가 30개 이상이다.
- 모든 민간 의뢰는 후보 수가 30개 이상이다.
- 생성 결과의 사유 수는 항상 1개 또는 2개다.
- 생성 결과의 사유에는 중복 문자열이 없다.
- 대표 공고의 그룹 공통 후보와 세부 후보가 모두 병합되는지 검증한다.
