# 주사위 굴리기 설계

## 목표

`@nihilapp/diceroll-v3`를 사용해 프리셋과 커스텀 주사위식을 실행하는 도구를 만든다. 화면 전체는 스크롤하지 않고, 각 탭의 굴림 이력만 독립적으로 스크롤한다.

## 범위

- `/apps/dice-roller` 조립 페이지와 주사위 굴리기 렌더링 컴포넌트를 추가한다.
- 프리셋 탭에는 개수 숫자 입력, 주사위 드롭다운, 굴리기 버튼을 제공한다.
- 커스텀 탭에는 주사위식 입력과 굴리기 버튼을 제공한다.
- 각 탭은 자신의 최신순 이력을 유지한다.
- 패키지 실행 오류는 현재 탭의 입력 영역에서 표시한다.

## 제외 범위

- 최소·최대 강제 굴림
- 이력의 브라우저 저장 또는 서버 저장
- 프리셋 편집 기능
- 라이브러리 문법을 화면에서 별도로 편집하거나 확장하는 기능

## 프리셋

프리셋은 `app/data/dice-roller-presets.data.ts`에서 UI 표기와 실행 식을 분리해 관리한다.

| UI 표기 | 실행 식 |
| --- | --- |
| D2 | `d2` |
| D4 | `d4` |
| D6 | `d6` |
| D8 | `d8` |
| D10 | `d10` |
| D12 | `d12` |
| D20 | `d20` |
| D100 | `d100` |
| DF | `dF` |
| D% | `d%` |

일반 주사위와 `DF`는 선택한 개수를 식의 앞에 붙인다. `D%`는 패키지에서 항상 한 번의 퍼센타일 굴림으로 처리하므로, 선택하면 개수 입력을 `1`로 설정하고 비활성화한다.

## 화면 구성

`DiceRoller`는 탭, 실행 영역, 이력 영역을 수직 `flex` 컨테이너로 구성한다.

```text
[ 프리셋 | 커스텀 ]

프리셋
[ 개수 ] [ D20 선택 ] | [ 굴리기 ]

커스텀
[ d20+5               ] | [ 굴리기 ]

[ 탭별 최신순 이력 ................................... ]
[ 이력만 세로 스크롤 .................................. ]
```

- 프리셋 탭의 상단은 `[숫자 인풋] [드롭다운] | [굴리기 버튼]`으로 둔다.
- 커스텀 탭의 상단은 `[주사위식 인풋] | [굴리기 버튼]`으로 둔다.
- 좁은 화면에서는 실행 조작이 줄바꿈될 수 있지만, 탭과 이력의 수직 구조는 유지한다.
- 이력 영역은 `flex-1 min-h-0 overflow-y-auto`로 두고, 화면 바깥 컨테이너는 `overflow-hidden`으로 둔다.
- 이력은 탭 전환 후에도 유지하지만, 페이지를 벗어나거나 새로고침하면 초기화한다.

## 데이터 흐름

1. 프리셋 탭은 개수와 선택 프리셋으로 실행 식을 만든다. 예: 개수 `2`, D6은 `2d6`이다.
2. 커스텀 탭은 입력 문자열을 그대로 사용한다.
3. 두 탭 모두 `rollDiceExpression()`을 호출한다.
4. 반환된 `DiceExpressionResult[]` 전체를 하나의 이력 항목으로 저장한다. 공백으로 여러 식을 실행한 경우에도 한 번의 사용자 실행으로 묶는다.
5. 새 이력 항목은 현재 탭의 배열 맨 앞에 추가한다.

## 결과 표시

- 이력 항목은 실행한 식과 총합을 우선 표시한다.
- 다중 식 실행은 이력 항목 안에서 결과별로 식과 총합을 표시한다.
- 상세 영역은 `rollDetails`와 `modifiers`로 구성한다.
- 기본 굴림의 눈, 유지·버림 굴림의 전체·유지·버림 눈, Fate의 눈, 퍼센타일 결과는 `RollKind`에 따라 안전하게 분기해 표시한다.
- 상세에 없는 결과 유형도 총합과 식은 항상 표시한다. 표현하지 못한 내부 필드를 억지로 가정하지 않는다.

## 오류 처리

- 빈 커스텀 식은 실행하지 않고 입력 아래에 오류를 표시한다.
- 라이브러리 호출 중 발생한 오류는 현재 탭의 오류 상태로 변환해 표시한다.
- 오류가 발생해도 기존 이력과 다른 탭의 상태는 변경하지 않는다.
- 입력값을 바꾸거나 성공적으로 굴리면 해당 탭의 이전 오류를 제거한다.

## 구현 경계

| 경로 | 책임 |
| --- | --- |
| `app/pages/apps/dice-roller.vue` | 렌더링 컴포넌트 조립만 담당 |
| `app/components/diceRoller/DiceRoller.vue` | 탭 상태, 실행 상태, 이력 상태 조합 |
| `app/components/diceRoller/*` | 프리셋 입력, 커스텀 입력, 이력, 결과 상세의 표현 책임 |
| `app/data/dice-roller-presets.data.ts` | 프리셋 정적 데이터 |
| `app/data/dice-roller.types.ts` | 이력과 프리셋 도메인 타입 |

모든 신규 렌더링 컴포넌트는 프로젝트의 `cva`와 `cn` 구조를 사용한다. 함수 이름은 `on<액션><대상>` 규칙을 따른다. 앱 소스 import는 `~/` 별칭을 사용한다.

## 검증

- 프리셋별 식 조합: 일반 주사위와 `DF`의 개수 반영, `D%`의 개수 고정
- 정상 굴림: 결과가 현재 탭의 최신 이력에 추가되는지
- 다중 식 커스텀 굴림: 하나의 이력 항목에 결과 배열이 보존되는지
- 오류 굴림: 오류가 표시되고 기존 이력이 유지되는지
- 탭 전환: 각 탭의 입력과 이력이 독립적으로 유지되는지
- 화면 확인: 페이지 바깥 스크롤 없이 이력 영역만 스크롤되는지
- 변경 파일 lint 및 관련 테스트, Nuxt build
