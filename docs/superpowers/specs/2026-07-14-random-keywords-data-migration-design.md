# 랜덤 키워드 데이터 이관 설계

## 목표

기존 랜덤 키워드 JSON의 유효 항목을 중복 없이 `app/data/random-keywords/`의 풀 단위 TypeScript 데이터 모듈로 이관한다.

## 범위

- 원본의 40개 키워드 풀을 각각 하나의 `<분류>.data.ts` 파일로 분리한다.
- 각 파일은 해당 풀의 표시명, 분류, 경로형 여부, 키워드 배열을 export한다.
- 통합 카탈로그는 모든 풀을 안정적인 식별자로 조회할 수 있게 한다.
- 중복 항목은 같은 풀 안에서는 한 번만 남긴다. 서로 다른 풀에 공통으로 존재하는 값은 각 풀의 의미를 보존하기 위해 유지한다.
- 인코딩이 깨진 항목은 실제 키워드 풀에서 제외하고 감사 데이터에 원문, 위치, 제외 사유를 남긴다.

## 구조

`app/data/random-keywords/`는 다음 네 층으로 구성한다.

1. `random-keyword.types.ts`: 풀의 공통 타입과 분류 식별자를 정의한다.
2. `<영역>-<풀>.data.ts`: 원본의 40개 풀을 하나씩 담는다.
3. `random-keywords-catalog.data.ts`: 개별 풀을 모아 앱이 탐색·무작위 선택에 사용할 전체 카탈로그를 export한다.
4. `random-keywords-audit.data.ts`: 중복 제거 통계와 깨진 문자열 제외 기록을 export한다.

원본의 슬래시 경로형 값은 문자열을 바꾸지 않고 유지한다. 이후 UI가 필요하면 슬래시를 기준으로 경로를 표시하거나 필터할 수 있다.

## 검증

- 카탈로그에 원본 40개 풀이 모두 등록됐는지 확인한다.
- 각 풀의 키워드가 중복 없이 유지되는지 확인한다.
- 모든 유효 원본 값이 이관됐고, 제외값은 감사 데이터에서만 확인되는지 검사한다.
- 깨진 문자 `�`가 실제 키워드 배열에 남지 않았는지 검사한다.
