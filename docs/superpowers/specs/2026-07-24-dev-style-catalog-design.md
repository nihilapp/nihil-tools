# 개발용 스타일 카탈로그 설계

## 목적

스토리북을 별도로 설치하지 않고도 현재 앱의 공용 UI 컴포넌트와 디자인 토큰을 한 화면에서 확인한다. 이 화면은 컴포넌트 사용자가 구현 결과와 각 변형을 빠르게 비교하는 개발용 카탈로그이며, 일반 도구 목록에는 노출하지 않는다.

## 경로와 접근 범위

- 경로는 `/dev/style`이다.
- `app/data/app-list.data.ts`에 도구로 등록하지 않는다.
- `app/pages/dev/style.vue`는 렌더링 컴포넌트만 조립한다.
- 실제 화면 구성은 `app/components/views/DevStyleCatalog.vue`가 담당한다.

## 화면 구조

화면 상단에는 페이지 제목과 설명, 그리고 앵커 이동 탭을 둔다. 탭은 `토큰`, `버튼`, `입력`, `선택`, `피드백`, `레이아웃` 순서이며 각 섹션으로 즉시 이동한다.

각 섹션은 `UiSection` 안에 두고, 비교 대상은 `UiSurface`로 감싼다. 예제에는 코드 문자열이나 가짜 목업을 쓰지 않고 실제 `Ui*` 컴포넌트를 렌더링한다. 따라서 공용 컴포넌트의 변경은 이 페이지에도 즉시 반영된다.

## 카탈로그 범위

### 토큰

- 캔버스, 표면, 잉크, 보조 잉크, 경계선, 주요 액션, 위험 상태 색상을 견본과 함께 표시한다.
- 기본 간격과 `rounded-2` 모서리 규칙을 작은 표본으로 보여준다.

### 버튼

- `primary`, `secondary`, `outline`, `ghost`, `danger` 변형을 모두 표시한다.
- 각 변형을 `sm`, `md` 크기로 나열한다.
- 비활성 및 로딩 상태를 별도 행으로 보여준다.

### 입력

- `UiInput`의 기본, 라벨, 플레이스홀더, 오류 상태를 표시한다.
- 여러 줄 텍스트용 `UiTextarea`를 추가하고 기본, 라벨, 오류 상태를 함께 표시한다.

### 선택

- `UiSelect`의 기본, 라벨, 비활성 선택지를 표시한다.
- 단일 선택 체크박스와 라디오 그룹을 공용 프리미티브로 추가하고, 기본·선택·비활성 상태를 표시한다.

### 피드백

- 필수 입력 오류와 비활성 컨트롤의 시각적 상태를 모은다.
- 성공 토스트나 모달처럼 아직 공용 계약이 없는 상위 컴포넌트는 이번 범위에서 제외한다.

### 레이아웃

- `UiSurface`와 `UiSection`의 기본 조합 및 중첩 예시를 표시한다.

## 컴포넌트 계약

- 모든 새 렌더링/UI 컴포넌트는 `cva`와 `cn` 패턴을 따른다.
- 공용 프리미티브의 `class` prop은 기존 방식처럼 기본값을 둔다.
- 입력 계열은 `v-model`을 지원하고, label 및 error를 통한 접근성 정보를 제공한다.
- 상호작용 함수는 `on<액션><대상>` 명명 규칙을 따른다.

## 검증

- 스타일 페이지의 섹션·버튼 변형·폼 상태 렌더링을 대상으로 하는 단위 테스트를 추가한다.
- 새 폼 프리미티브의 `v-model`, 오류, 비활성 상태를 단위 테스트한다.
- `pnpm run lint`, `pnpm test`, `pnpm build`로 전체 확인한다.

## 제외 범위

- Storybook 설치 및 설정
- 일반 도구 목록 노출
- 다크 모드, 테마 전환기, 토스트·모달·데이터 테이블 등 별도 상호작용 계약이 필요한 고수준 컴포넌트
