# 랜덤 키워드 생성기 UI 설계

## 목표

리뉴얼 키워드 23,559개를 상단 카테고리 기반으로 추첨하고, 캐릭터 배경 스토리는 별도 화면에서 한 문장으로 생성한다.

## 화면 경계

- `/apps/random-keyword-generator`: 일반 랜덤 키워드 생성기
- `/apps/random-keyword-generator/background-story`: 캐릭터 배경 스토리 생성기
- 두 화면 모두 기존 `UiButton`과 프로젝트 색상 토큰을 사용한다.
- 페이지 파일은 렌더링 컴포넌트를 조립만 한다.

## 일반 키워드 생성기

상단 카테고리는 `전체`, `자연·환경`, `사물·물질`, `문화·신앙·신화`, `건강·질병`, `추상·상태·능력`, `인물·캐릭터`, `판타지 존재`, `수식어`다. 선택한 대분류의 세부 테마는 보조 필터로 표시한다.

생성 수량은 `1`, `5`, `10`만 제공하며 기본값은 `1`이다. 특정 대분류는 그 안의 모든 키워드에서 균등 추첨한다. `전체`는 대분류를 먼저 균등 추첨한 뒤 해당 대분류에서 추첨하여 의류·수식어 대량 데이터가 결과를 지배하지 않게 한다.

한 번의 결과에서는 동일한 `themeId + keyword` 조합을 반복하지 않는다. 표현이 같아도 테마가 다르면 보존하며, 결과 항목에는 대분류와 세부 테마 경로를 표시한다.

## 캐릭터 배경 스토리 생성기

생성 수량 선택은 없으며 한 번에 결과 문장 하나만 생성한다.

문장 형식은 아래로 고정한다.

```text
<목적>를 위해 <출신지>에서 온 <누구>
```

- 현실: 현실 목적 + 현실 출신지 + 인물 직업
- 판타지: 판타지 목적 + 판타지 출신지 수식 + 판타지 출신지 장소 + 종족 및 직업
- 공용: 현실·판타지 목적과 출신지 재료를 모두 후보로 사용하는 통합 조합

`<누구>`는 기본적으로 직업을 사용한다. 판타지에서는 종족과 직업을 결합하며, 종족이 없는 경우 직업만 사용한다.

## 데이터 접근과 선택기

UI는 61개 테마 모듈을 직접 순회하지 않는다. UI용 대분류 설정 데이터가 세부 테마 ID 목록을 소유하고, 선택 유틸리티가 테마의 키워드 배열을 평탄화한다. 메타데이터 필드는 제외하고 중첩 배열의 문자열만 추출한다.

배경 스토리 선택기도 별도 유틸리티로 두며, 선택에 필요한 난수 함수를 인자로 받아 단위 테스트에서 결과를 재현할 수 있게 한다.

## 제외 범위

- 캐릭터 배경 스토리 데이터는 일반 생성기의 `전체` 및 모든 일반 대분류에서 제외한다.
- 서버 저장, 결과 이력 저장, 복사 기능, 사용자 정의 데이터 편집은 이번 범위에 포함하지 않는다.
