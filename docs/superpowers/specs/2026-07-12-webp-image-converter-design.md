# WebP 이미지 변환기 설계

## 목표

`nihil-tools`에 PNG와 JPG/JPEG 파일을 WebP로 변환하는 로컬 이미지 변환 앱을 추가한다. 사용자는 여러 파일을 드래그 앤 드롭 또는 파일 선택으로 추가하고, 한 번의 작업으로 모두 변환한 뒤 결과를 내려받을 수 있다.

## 범위

- 앱 목록에 `이미지 WebP 변환`을 추가한다.
- PNG, JPG, JPEG 파일을 여러 개 선택하거나 드래그 앤 드롭으로 추가한다.
- 변환 품질을 40~100% 범위에서 조절하며 기본값은 85%로 한다.
- 대기·변환 중·완료·오류 상태를 파일별로 표시한다.
- 현재 대기 또는 오류 파일을 순서대로 일괄 변환한다.
- 완료 파일은 개별 다운로드할 수 있으며, 일괄 다운로드는 각 파일의 다운로드를 순서대로 시작한다.
- 파일 삭제, 전체 목록 비우기, 원본 및 결과 용량과 절감률 표시를 제공한다.

## 제외 범위

- 파일 서버 업로드와 저장
- ZIP 파일 생성
- 이미지 크기 조절, 자르기, 회전 등 변환 외 편집

## 구조

페이지는 전용 렌더링 컴포넌트만 조립한다. 렌더링 컴포넌트는 업로드 영역, 변환 옵션 및 요약, 변환 목록을 구성한다. 파일 형식 판별, 출력 파일명 생성, 용량 표시, Canvas 변환은 별도 유틸리티로 분리해 테스트한다.

기존 `trpg-tools`의 `PngToWebpConverter.vue` 흐름을 참고한다. 브라우저의 Canvas `toBlob` API로 `image/webp` Blob을 생성하고, 원본과 결과의 Object URL을 항목 삭제 및 컴포넌트 해제 시 모두 해제한다.

## 처리 흐름

1. 사용자가 파일을 추가한다.
2. PNG/JPG/JPEG만 변환 목록에 대기 상태로 추가한다.
3. 사용자가 품질을 정하고 일괄 변환을 실행한다.
4. 목록의 대기 또는 오류 항목을 하나씩 Canvas로 WebP Blob으로 변환한다.
5. 각 결과를 다운로드 가능한 Object URL로 만들고, 완료 또는 오류 상태를 기록한다.
6. 사용자는 완료된 파일을 개별 또는 일괄 다운로드한다.

## 오류 처리

- 지원하지 않는 파일은 목록에 추가하지 않는다.
- 이미지를 읽지 못하거나 Canvas/WebP 변환을 지원하지 않으면 해당 항목만 오류 상태로 표시한다.
- 오류 항목은 품질을 바꾼 뒤에도 다시 일괄 변환할 수 있다.

## 검증

- 유틸리티 단위 테스트로 입력 형식 판별, 출력명 생성, 용량 형식화, 절감률 계산을 검증한다.
- UI 테스트로 파일 추가, 상태별 일괄 변환 대상 선택, 완료 항목 다운로드 동작을 검증한다.
- 린트, 테스트, 프로덕션 빌드로 통합 확인한다.
